# Slack Protocol — General

## Channels

| Channel | ID | Purpose |
|---|---|---|
| #general | `C0A9L8KDFEW` | Cross-team announcements and updates |
| #standup | `C0AJNMCSTT2` | Daily standups |
| #news | `C0AHN2684EP` | External news relevant to Ametyst |

## When to post

### #general (`C0A9L8KDFEW`)
- Cross-team announcements that affect both Governance and Strategy
- Protocol updates
- Company-wide status updates

### #standup (`C0AJNMCSTT2`)
- Daily standup reports

### #news (`C0AHN2684EP`)
- External news, market intel, competitor updates

## Message formats

### #general — Protocols update
```
:books: *Protocols — [action]*
Entry: [doc name]
Category: [category]
[Optional: summary of content]
```

### #general — Cross-team announcement
```
:mega: *Announcement*
[Content]
```

### #standup
```
*Standup — [Name] — [Date]*
- Yesterday: [done]
- Today: [planned]
- Blockers: [any]
```

### #news
```
:newspaper: *[Source] — [Headline]*
[Summary]
Why it matters: [relevance to Ametyst]
```

## Rules
1. Always use channel IDs, never channel names
2. Use threads for follow-up discussion
3. Check recent history before posting duplicate updates
4. Language: English
5. #standup format is strict — follow the template exactly
